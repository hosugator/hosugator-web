# ElastiCache

## 본질 (Essence)
데이터베이스 위에 위치하여 자주 조회되는 데이터를 메모리에 저장함으로써 응답 속도를 높이고 원본 DB의 읽기 부하를 줄이는 완전 관리형 인메모리 캐시 서비스

## 구성 요소 (Sub-items)
### 1. Redis (Remote Dictionary Server)
가장 널리 사용되는 엔진으로 단순 캐시 이상의 기능을 제공함. 리스트, 세트, 정렬된 세트 등 다양한 데이터 구조를 지원하며 멀티 AZ를 통한 자동 장애 조치와 읽기 복제본 생성이 가능함. 스냅샷을 통한 백업과 복구 기능을 지원하여 고가용성이 필요한 환경에 적합함.

### 2. Memcached
순수 인메모리 키-값 저장소로 설계된 단순하고 빠른 엔진임. 멀티스레드 아키텍처를 지원하여 대규모 노드 확장에 유리함. 다만 복제나 스냅샷 기능이 없으므로 데이터 유실 시 재구성이 용이하고 고가용성보다 단순 캐싱 성능이 중요한 환경에 적합함.

### 3. Serverless
사용자가 노드 크기나 개수를 직접 관리하지 않고 애플리케이션의 사용량에 따라 컴퓨팅 및 메모리 용량이 자동으로 조절되는 옵션임. 운영 오버헤드를 최소화하고 데이터 저장량 및 사용량에 따라 비용을 지불함.

## 구조 (Mechanism)
사용자가 데이터를 요청하면 애플리케이션은 먼저 ElastiCache를 확인한 후 데이터가 있으면 즉시 반환하고 없으면 원본 DB에서 데이터를 읽어 캐시에 저장한 뒤 반환함. 이를 통해 DB 도달 횟수를 줄여 전체적인 시스템 지연 시간을 밀리초 미만으로 단축함.

## 확장 (Connection)
데이터베이스 읽기 성능 최적화를 위해 Read Replica와 함께 고려되지만 읽기 횟수 자체를 차단하는 데는 캐싱이 더 효과적임. 세션 관리, 실시간 순위표, 메시지 브로커 등의 용도로도 확장 가능함.
