# AWS Organizations

## 본질 (Essence)
여러 개의 독립된 집(계정)들을 하나의 단지로 묶어 관리하고, 단지 전체에 적용되는 공동 주택 관리 규약을 배포하는 중앙 관리실

## 구조 (Mechanism)
- **정의**: 다수의 AWS 계정을 중앙에서 통합 관리하고 거버넌스를 구현하기 위한 계정 관리 서비스입니다.
- **핵심**: 
    - **통합 빌링(Consolidated Billing)**: 여러 계정의 사용량을 합산하여 결제함으로써 볼륨 할인을 받고 비용을 한곳에서 관리합니다.
    - **SCP(Service Control Policies)**: 최상위 보안 가이드라인을 설정하여 하위 계정의 Root 사용자조차 특정 서비스를 사용하지 못하도록 강력하게 제한합니다.
    - **계정 자동 생성**: 콘솔 조작 없이 API를 통해 새로운 환경(개발/운영 등)의 계정을 즉시 생성하고 표준 보안 설정을 적용합니다.

## 확장 (Connection)
- **연결**: 개별 계정의 보안 설정이 각 집의 잠금장치라면, Organizations의 SCP는 아파트 단지 전체의 가스 공급을 차단하거나 통행을 제한하는 중앙 통제 시스템과 유사함
- **응용**: 기업 규모가 커짐에 따라 개발, 검증, 운영 계정을 물리적으로 분리하고 중앙 보안 팀이 전체적인 가이드레일을 유지하는 데 사용함
