# AWS Transfer Family

## 요약 (Summary)
외부 사용자가 보낸 데이터(SFTP·FTPS·FTP)를 객체 데이터 형태로 변환하여 S3나 EFS로 바로 전송해주는 완전 관리형 서버리스 서비스이다.

## 배경 (Background)
- 탄생 배경: 외부 파트너와 파일을 주고받을 때 기존의 표준 프로토콜(SFTP 등)을 유지해야 하지만, 이를 위해 직접 서버를 구축하고 관리하는 것은 보안 및 운영상 부담이 큼
- 핵심 과제: 별도의 서버 인프라 관리 없이 외부 파트너가 던져주는 파일을 안전하게 수신하여 즉시 클라우드 스토리지에 저장해야 함

## 솔루션 (Solution)
- 핵심 설계: AWS가 관리하는 서비스형 엔드포인트를 제공하여 외부 프로토콜과 내부 스토리지(S3/EFS)를 1:1로 중계함
- 작동 메커니즘: 외부인이 SFTP 클라이언트로 접속하여 파일을 전송하면 -> 관리형 서버가 이를 수신하여 지정된 S3 버킷에 객체 형태로 자동 저장함

## 변별점 (Distinction)
- 비유: 외부인이 우리 집 창고(S3)에 물건을 넣으려 할 때, 내가 직접 문을 지키는 대신 보안 요원이 상주하며 물건만 받아주는 '전용 무인 택배 보관함'을 입구에 설치하는 것과 같음
- 대안과의 차이: 
	 - [S3 File Gateway]: 내부 시스템의 저장소 확장 목적이 큰 반면, Transfer Family는 외부 협력사와의 안전하고 간편한 데이터 수집(Ingestion)에 특화됨
