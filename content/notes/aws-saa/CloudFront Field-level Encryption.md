# CloudFront Field-level Encryption

## 본질 (Essence)
> 데이터가 서버라는 '은행'에 도착하기 전, 현관문(Edge)에서부터 특정 담당자만 열 수 있는 별도의 '개별 금고'에 민감 정보만 따로 담아 잠그는 이중 잠금 장치

## 구조 (Mechanism)
- **정의**: 사용자가 제출한 민감한 데이터(필드)를 CloudFront Edge에서 공개키로 암호화하여, 전체 애플리케이션 스택 내내 암호화된 상태를 유지하고 오직 사설키를 가진 최종 목적지에서만 해독하게 하는 보안 기능
- **핵심**: HTTPS라는 '안전한 통로'를 제공하는 것을 넘어, 데이터 '알맹이' 자체를 암호화함으로써 중간 계층이나 서버 메모리에서 평문 데이터가 노출되는 문제를 근본적으로 해결함

## 확장 (Connection)
- **연결**: [[HTTPS/TLS]]가 도로 위의 무장 수송차라면, [[Field-level Encryption]]은 그 차 안에 실린 개별 잠금 상자와 같음
- **응용**: 신용카드 번호, 주민등록번호 등 법적 규제가 엄격한 개인 식별 정보를 클라우드 전 구간에서 안전하게 보호해야 하는 금융 및 의료 서비스 설계에 필수적으로 활용
