# AWS CodeCommit
## 정의
AWS에서 호스팅되는 완전 관리형 소스 제어 서비스로, 안전하고 확장성이 뛰어난 프라이빗 Git 리포지토리를 지원
## 원리
### Git 기반 중앙 집중식 저장소
- 표준 Git 프로토콜을 사용하므로, 개발자는 기존 Git 클라이언트 및 도구를 그대로 사용할 수 있다.
- 코드, 바이너리, 라이브러리 등 모든 유형의 데이터를 프라이빗 환경에 저장하고 버전 관리한다.
### 높은 내구성과 가용성
- 리포지토리의 데이터는 [[S3]]에 암호화되어 저장되며, 다중 가용 영역([[AZ]])에 걸쳐 중복 저장되어 높은 내구성과 가용성을 제공한다.
## 요소
- 리포지토리(Repository): 소스 코드 및 파일의 컬렉션이다. 각 리포지토리는 Git 기반의 버전 제어 시스템을 사용한다.
- 브랜치(Branch): 코드의 독립적인 개발 라인을 유지하는 데 사용된다.
- 풀 리퀘스트(Pull Request): 한 브랜치에서 다른 브랜치로 코드를 병합(Merge)하기 전에 코드 검토(Code Review)를 요청하는 프로세스이다.
- 승인 규칙(Approval Rule): 코드를 병합하기 위해 특정 수의 사용자나 [[IAM]] 사용자의 승인이 필요하도록 설정하는 기능이다.
## 특징
- 완전 관리형: 사용자가 자체 서버를 설치, 운영, 스케일링, 패치할 필요가 없다.
- 보안: 저장 중인 데이터와 전송 중인 데이터 모두 암호화되며, 접근 제어는 AWS IAM을 통해 세밀하게 관리된다.
- 통합: [[CodePipeline]] 및 [[CodeBuild]]와 같은 AWS 개발자 도구뿐만 아니라, JIRA, Jenkins와 같은 타사 도구와도 쉽게 통합된다.
- 알림: 리포지토리에 이벤트가 발생하면 [[SNS]]를 통해 개발팀에 알림을 보낼 수 있다.
## 비교
| 구분    | CodeCommit (AWS Managed)   | GitHub/GitLab (Public/Self-Managed)   |
| :---- | :------------------------- | :------------------------------------ |
| 관리 주체 | AWS가 완전 관리                 | SaaS (GitHub/GitLab) 또는 사용자가 자체 서버 관리 |
| 접근 제어 | AWS IAM 역할 및 사용자 사용        | 자체 사용자 계정 및 권한 모델 사용                  |
| 비용    | 월별 활성 사용자 및 스토리지에 따라 요금 부과 | 다양한 요금제                               |
## 예시
- CI/CD 파이프라인의 시작점:
    1. 개발자가 로컬 PC에서 코드를 작성하고 CodeCommit 리포지토리에 Git Push 한다.
    2. 이 Push 이벤트는 [[CodePipeline]]을 트리거하여 빌드 프로세스를 시작한다.
    3. [[CodePipeline]]은 [[CodeBuild]]를 호출하여 코드를 컴파일하고 테스트를 실행한다.
    4. 빌드가 성공하면 [[CodeDeploy]]를 사용하여 코드를 [[EC2]] 또는 [[Lambda]]와 같은 환경에 배포한다.
    5. CodeCommit은 전체 CI/CD 파이프라인의 안정적인 소스 코드 저장소 역할을 수행한다.
