# Comparison - AWS Budgets vs Cost Explorer vs CUR

## 비교 목적 (Objective)
클라우드 비용 관리의 목적(알람, 분석, 기록)에 따른 최적의 AWS 도구 선정.

## 요소별 상세 비교 (Feature Matrix)
| 비교 기준 | [[AWS Budgets]] | [[Cost Explorer]] | [[Cost & Usage Report]] |
| :--- | :--- | :--- | :--- |
| 핵심 본질 | 비용 임계치 기반 알람 도구 | 비용 트렌드 시각화 분석 도구 | 상세 비용 내역 원천 데이터 제공 |
| 장점 (Pros) | 즉각적인 알림 및 자동 대응 가능 | 그래프를 통한 직관적 패턴 파악 | 가장 세밀한 수준의 데이터 분석 |
| 단점 (Cons) | 세밀한 원인 분석에는 한계가 있음 | 실시간 알람 기능이 약함 | 분석을 위해 별도 도구(Athena) 필요 |
| 비용/관리 | 매우 저렴 / 관리 거의 없음 | 무료(일부 기능 유료) / 관리 낮음 | 스토리지 및 쿼리 비용 발생 / 관리 높음 |

## 선택 기준 및 로직 (Selection Logic)

### AWS Budgets가 적합한 경우
- 조건 (IF): 특정 서비스나 계정의 지출이 계획된 예산을 초과할 때 즉시 알림을 받아야 할 때
- 이유 (THEN): SNS 연동을 통한 자동 알람 기능이 내장되어 있어 가장 적은 비용과 노력으로 목표 달성 가능함

### Cost Explorer가 적합한 경우
- 조건 (IF): 과거 지출 패턴을 분석하여 미래 비용을 예측하거나 낭비 요소를 시각적으로 찾고 싶을 때
- 이유 (THEN): 별도의 설정 없이 대시보드에서 서비스별, 태그별 비용 통계를 즉시 확인할 수 있음

### Cost & Usage Report가 적합한 경우
- 조건 (IF): 기업 내 회계 시스템과 연동하거나, 비즈니스 인텔리전스(BI) 도구로 초정밀 분석이 필요할 때
- 이유 (THEN): AWS에서 제공하는 가장 상세한 데이터를 S3에 저장하므로, 외부 도구와의 결합 및 대규모 데이터 처리에 유리함

---
 Conclusion: 알람은 Budgets, 분석은 Cost Explorer, 데이터 추출은 CUR
