# Comparison: FTP vs FTPS vs SFTP

## 비교 목적 (Objective)
보안 요구 수준과 인프라 호환성에 따른 최적의 파일 전송 프로토콜 선정

## 요소별 상세 비교 (Feature Matrix)
| 비교 기준 | [[FTP]] | [[FTPS]] | [[SFTP]] |
| :--- | :--- | :--- | :--- |
| 핵심 본질 | 암호화가 없는 표준 전송 규약 | SSL/TLS 암호화가 추가된 FTP | SSH 보안 터널 기반 전송 규약 |
| 장점 (Pros) | 설정이 매우 간단하고 빠름 | 기존 FTP 환경에서 보안 강화 | 높은 보안성 및 단일 포트 관리 |
| 단점 (Cons) | 데이터 탈취에 매우 취약함 | 다중 포트 사용으로 방화벽 복잡 | SSH 설정에 대한 지식 필요 |
| 비용/관리 | 별도 관리 요소 없음 | 인증서 및 방화벽 관리 필요 | SSH 키 및 서버 보안 관리 필요 |

## 선택 기준 및 로직 (Selection Logic)

### FTP가 적합한 경우
- 조건 (IF): 신뢰할 수 있는 내부 네트워크에서 단순 테스트용으로 파일을 전송할 때
- 이유 (THEN): 보안보다 구축 속도와 호환성이 최우선이므로 가장 단순한 규약을 사용함

### FTPS가 적합한 경우
- 조건 (IF): 반드시 기존의 FTP 클라이언트 환경을 유지하면서 전송 구간 암호화만 추가해야 할 때
- 이유 (THEN): SSL/TLS 인증서를 통해 데이터를 보호하면서도 익숙한 FTP 인터페이스를 그대로 활용할 수 있음

### SFTP가 적합한 경우
- 조건 (IF): 인터넷망을 통해 외부와 파일을 주고받으며 보안과 운영 효율이 중요할 때
- 이유 (THEN): 강력한 암호화와 단일 포트(22) 사용으로 관리 오버헤드를 줄이면서 보안 사고를 예방함

---
 Conclusion: 단순 전송은 FTP, 호환 중심 보안은 FTPS, 현대적 표준 보안은 SFTP
