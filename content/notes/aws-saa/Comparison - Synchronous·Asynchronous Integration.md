# Synchronous vs Asynchronous Integration (통합 방식 비교)
## 정의
분산 시스템의 서로 다른 구성 요소들이 통신하고 데이터를 교환하는 방식인 동기적 통합과 비동기적 통합의 구조, 성능, 복원력 측면에서의 차이점을 비교 분석하는 개념
## 요소
### Synchronous Integration (동기적 통합)
요청을 보낸 서비스(Client)가 응답을 받을 때까지 실행을 멈추고 기다리는 통신 방식이다. (예: HTTP/S 기반의 RESTful API 호출)
### Asynchronous Integration (비동기적 통합)
요청을 보낸 서비스가 응답을 기다리지 않고 즉시 다음 작업을 계속하며, 응답이나 결과를 나중에 별도의 메커니즘을 통해 받는 통신 방식이다. (예: SQS, SNS, Lambda Event Invocation)
### Intermediate Buffer (중간 버퍼)
비동기 통신에서 요청과 응답 사이에 발생하는 시간차를 관리하기 위해 사용하는 대기열(Queue) 또는 스트림(Stream)이다. (예: SQS, Kinesis)
## 원리
### Coupling Level (결합 수준)
- 동기적: 요청을 받은 서비스(Server)의 가용성이 클라이언트의 요청 처리 능력에 직접적인 영향을 미친다. (높은 결합도)
- 비동기적: 중간 버퍼를 사용해 클라이언트와 서버가 간접적으로 통신하므로, 한 쪽의 장애가 다른 쪽에 영향을 미치지 않는다. (낮은 결합도/Decoupling)
### Failure Handling (장애 처리)
- 동기적: 서버에 장애가 발생하거나 지연이 발생하면, 클라이언트는 타임아웃 오류를 받으며 전체 트랜잭션이 실패한다.
- 비동기적: 클라이언트는 요청(메시지)을 버퍼에 안전하게 전달하는 즉시 성공 응답을 받는다. 백엔드 서비스가 나중에 메시지를 처리하므로 복원력이 높다.
### Scaling (크기 조정)
- 동기적: 클라이언트와 서버가 같은 속도로 확장해야 하므로 개별적인 확장이 어렵다.
- 비동기적: 생산자(Client)는 빠르게 확장하고, 소비자(Server)는 대기열의 메시지 양에 따라 자신의 속도와 용량에 맞춰 독립적으로 확장 및 축소할 수 있다.
## 특징
### Latency (지연 시간)
- 동기적: 낮다. 요청과 응답이 즉시 이루어지므로 최종 사용자의 응답 시간이 짧다.
- 비동기적: 높다. 최종 결과가 나올 때까지 시간이 걸리지만, 클라이언트의 즉각적인 응답(Acknowledgement)은 빠르다.
### Resiliency (복원력)
비동기 통합은 중간 버퍼(SQS)가 메시지를 안전하게 보관하고, 소비자가 실패할 경우 재시도(Retry) 메커니즘을 적용할 수 있으므로, 동기적 통합 대비 월등히 높은 복원력을 제공한다.
### Use Case (사용 사례)
- 동기적: 사용자 로그인, 실시간 정보 조회(예: 환율 조회), 프론트엔드 API 호출 등 즉각적인 결과가 필요한 작업에 적합하다.
- 비동기적: 주문 처리, 대용량 파일 업로드 후 처리, 이메일 발송 등 응답을 즉시 기다릴 필요가 없는 장기 실행 작업에 적합하다.
## 비교
| 구분 | Synchronous Integration (동기적) | Asynchronous Integration (비동기적) |
| :--- | :--- | :--- |
| 통신 형태 | 요청-응답(Request-Response) | 메시지 기반(Message-based) |
| 결합 수준 | 높음 (타 서비스 가용성에 종속됨) | 낮음 (Decoupling 구현) |
| 핵심 이점 | 낮은 지연 시간 및 즉각적인 피드백 | 높은 복원력 및 독립적인 확장 |
| 주요 서비스 | API Gateway, ALB (Target Group), EC2 직접 호출 | SQS, SNS, Lambda, Kinesis, EventBridge |
| 장애 시 영향 | 즉시 실패(Failure) | 메시지 버퍼링 후 재시도(Retry) 가능 |
## 예시
### 비동기적 주문 처리 성공 사례 (SQS 활용)
1. 사용자 요청: 사용자가 웹사이트에서 주문을 클릭한다.
2. 동기적 처리: 웹 서버가 SQS 대기열로 메시지를 보내고(Send), 즉시 사용자에게 "주문이 접수되었습니다"라고 응답한다. (지연 시간 100ms 미만)
3. 비동기적 처리: 주문 처리 백엔드 서비스(Consumer)가 대기열에서 메시지를 가져가 결제, 재고 처리(5초 소요)를 수행한다.
4. 결과: 사용자 경험 저하 없이 백엔드 작업을 안정적이고 복원력 있게 완료한다.
