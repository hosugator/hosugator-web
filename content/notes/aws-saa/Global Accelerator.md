# AWS Global Accelerator
## 정의
AWS 글로벌 네트워크 인프라를 사용하여 사용자 트래픽을 최적의 AWS 리전 애플리케이션 엔드포인트로 유도함으로써 애플리케이션의 가용성과 성능을 개선하는 네트워킹 서비스

## 원리
### 고정된 Anycast IP
- 액셀러레이터(Accelerator)는 전 세계의 AWS 엣지 로케이션에서 광고되는 두 개의 고정된 전역 Anycast IP 주소를 제공한다.
- 이 IP는 사용자가 접근하는 진입점 역할을 하며, 사용자는 가장 가까운 엣지 로케이션으로 라우팅된다.

### AWS 글로벌 네트워크 활용
- 사용자의 트래픽이 가장 가까운 엣지 로케이션에 도달한 후에는, 공용 인터넷의 혼잡 구간을 우회하여 지연 시간이 낮은 AWS 백본 네트워크를 통해 최종 애플리케이션 엔드포인트(예: [[ALB]], [[EC2]])로 전송된다.

## 요소
- 액셀러레이터(Accelerator): 고정된 진입점 IP 주소(Anycast IP)를 제공하는 기본 엔티티이다.
- 리스너(Listener): 인바운드 연결 요청을 처리할 포트 및 프로토콜(TCP 또는 UDP)을 정의한다.
- 엔드포인트 그룹(Endpoint Group): 트래픽을 수신할 하나 이상의 AWS 리전에 있는 엔드포인트([[ELB]], [[EC2]] 등)의 논리적 그룹이다.
- 엔드포인트: 트래픽의 실제 최종 대상 리소스이다.

## 특징
- 고정된 진입점: 고정된 IP 주소를 사용하여 클라이언트 측의 캐싱 문제나 DNS TTL(Time-to-Live) 문제 없이 트래픽 라우팅을 단순화한다.
- 성능 향상: AWS 글로벌 네트워크의 최적화된 경로를 사용하여 지연 시간을 공용 인터넷 대비 최대 60%까지 줄인다.
- 즉각적인 장애 조치: 엔드포인트 상태 확인을 수행하고, 문제가 감지되면 DNS 업데이트를 기다릴 필요 없이 자동으로 트래픽을 다른 리전의 건전한 엔드포인트로 전환한다.
- Non-HTTP/S 지원: [[CloudFront]]와 달리, TCP 및 UDP 트래픽(예: 게임, IoT)에 대해서도 가속화를 제공한다.

## 비교
| 구분         | Global Accelerator                  | [[CloudFront]]                    |
| :--------- | :---------------------------------- | :-------------------------------- |
| 최적화 대상 | 네트워크 경로 (최적의 리전으로 라우팅)          | 콘텐츠 캐싱 (엣지에서 콘텐츠 제공)          |
| 프로토콜   | TCP, UDP (모든 포트)                    | 주로 HTTP, HTTPS                    |
| 사용 사례  | API, 게임, IoT, 라이브 비디오 스트리밍 등 동적 트래픽 | 정적 웹사이트, 이미지, 비디오 파일 등 캐싱 가능한 콘텐츠 |

## 예시
- 글로벌 멀티플레이어 게임 애플리케이션:
    1. 게임 서버를 미국 동부와 유럽의 두 리전에 배포하고, 각 리전의 [[NLB]]를 Global Accelerator의 엔드포인트로 등록한다.
    2. 사용자는 전 세계 어디서든 Global Accelerator의 고정된 IP 주소로 접속을 시도한다.
    3. Global Accelerator는 각 사용자에게 가장 지연 시간이 낮은(Low Latency) AWS 엣지 로케이션을 통해 트래픽을 수신한 후, 최적의 AWS 글로벌 네트워크 경로를 통해 가까운 리전의 게임 서버로 전송한다.
    4. 이를 통해 전 세계 플레이어는 지연 시간이 최소화된 환경에서 게임을 플레이할 수 있다.
