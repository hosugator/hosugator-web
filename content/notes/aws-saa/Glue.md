# AWS Glue
## 정의
분석, 기계 학습 및 애플리케이션 개발을 위한 데이터를 쉽고 비용 효율적으로 준비하고 로드할 수 있는 서버리스 데이터 통합 서비스
## 원리
### ETL 작업 자동화
- 데이터 소스에서 데이터를 추출(Extract)하고, 필요한 형태로 변환(Transform)하며, 최종 목적지(Load)로 로드하는 과정을 서버리스 환경에서 자동으로 수행한다.
- Spark와 같은 분산 처리 프레임워크를 기반으로 ETL 작업을 실행하며, 인프라를 관리할 필요가 없다.
### 메타데이터 관리
- 데이터 카탈로그(Data Catalog) 기능을 통해 [[S3]]와 같은 데이터 레이크 내의 데이터 위치, 스키마, 런타임 메트릭을 저장하고 관리한다.
- 이는 [[Athena]] 및 [[Redshift]] Spectrum과 같은 다른 서비스가 데이터를 쿼리할 수 있도록 하는 핵심 구성 요소이다.
## 요소
- 데이터 카탈로그(Data Catalog): AWS Glue의 영구적인 메타데이터 저장소로, 데이터베이스와 테이블 스키마 정의를 저장한다.
- 크롤러(Crawler): 데이터 소스(예: [[S3]] 폴더)를 스캔하고 데이터의 스키마를 유추하여 Data Catalog에 메타데이터를 자동으로 기록한다.
- ETL 작업(Job): 데이터 변환 및 로드 작업을 수행하는 코드(Python 또는 Scala)이다. AWS Glue는 이 작업을 실행하는 서버를 관리한다.
- 개발 엔드포인트(Development Endpoint): 개발자가 ETL 스크립트를 대화식으로 작성하고 테스트할 수 있도록 제공되는 환경이다.
## 특징
- 서버리스: 인프라 프로비저닝이나 확장 관리가 필요 없으며, 실행 시간에 대해서만 비용을 지불한다.
- 데이터 카탈로그 통합: [[Athena]], [[Amazon EMR]], [[Redshift]] Spectrum 등이 Glue Data Catalog를 사용하여 데이터를 쿼리할 수 있다.
- 스키마 추론: 크롤러가 자동으로 스키마를 감지하여 ETL 작업 시간을 단축시킨다.
- Glue Studio: 시각적 인터페이스를 제공하여 코딩 없이 ETL 워크플로우를 쉽게 생성, 실행 및 모니터링할 수 있다.
## 비교
| 구분 | AWS Glue | [[Amazon EMR]] |
| :--- | :--- | :--- |
| 관리 방식 | 서버리스 (인프라 관리 불필요) | 클러스터 관리 필요 (하드웨어, OS, 소프트웨어) |
| 유연성 | ETL 및 카탈로그 기능에 중점 | Hadoop, Spark 등 다양한 빅데이터 프레임워크에 대한 높은 유연성 제공 |
| 비용 | 작업 실행 시간에 따라 요금 부과 | 클러스터 실행 시간에 따라 요금 부과 |
## 예시
- 데이터 레이크 준비:
    1. 사용자의 웹 로그 파일이 매일 [[S3]]의 원시(Raw) 폴더에 쌓인다.
    2. Glue 크롤러를 실행하여 이 로그 파일의 위치와 스키마를 유추하고 Glue 데이터 카탈로그에 등록한다.
    3. Glue ETL 작업을 생성하여 원시 로그 데이터를 읽어 들인다.
    4. 이 작업에서 데이터를 정리하고(Transform), 분석에 효율적인 Parquet 형식으로 변환한 후, [[S3]]의 정제(Curated) 폴더에 저장(Load)한다.
    5. 이제 [[Athena]]는 Glue 데이터 카탈로그를 사용하여 정제된 데이터를 빠르고 효율적으로 쿼리할 수 있다.
