# High-Performing Architecture Design (고성능 아키텍처 설계)
## 정의
컴퓨팅, 스토리지, 데이터베이스, 네트워킹 등 주요 아키텍처 계층에서 워크로드의 성능을 최대화하고 수요에 맞게 확장할 수 있도록 AWS 솔루션을 설계하는 방법론

## 요소

- Computing Layer (컴퓨팅 계층)
- Storage Layer (스토리지 계층)
- Database Layer (데이터베이스 계층)
- Networking Layer (네트워킹 계층)
- Data Collection and Transformation (데이터 수집 및 변환)

## 원리
- Data-Centric Approach: 아키텍처 패턴 및 구현을 선택할 때 데이터 중심 접근 방식을 사용하여 성능과 비용 효율성을 동시에 달성한다.
- Cost-Effective Solutions: 고성능을 달성하는 동시에 사용 리소스에 대한 비용을 최적화하는 솔루션을 선택한다.
- Workload Adaptability: AWS 서비스가 애플리케이션 요구에 맞게 자동으로 확장 및 축소될 수 있도록 설계하여, 용량 추측을 불필요하게 만든다.
