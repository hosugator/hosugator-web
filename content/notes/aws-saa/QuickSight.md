# Amazon QuickSight
## 정의
클라우드 기반의 확장 가능한 서버리스 비즈니스 인텔리전스(BI) 서비스

## 원리
### 데이터 연결 및 분석
- [[S3]], [[Redshift]], [[Athena]], [[Amazon RDS]] 등 다양한 AWS 데이터 소스 및 외부 데이터 소스에 연결하여 데이터를 추출한다.
- SPICE(Super-fast, Parallel, In-memory Calculation Engine) 엔진을 사용하여 데이터를 메모리에 저장하고 병렬 처리하여 대규모 데이터 세트에 대한 빠른 계산과 렌더링 속도를 제공한다.

### 자동화된 통찰
- ML Insights 기능을 사용하여 데이터 내의 이상 징후, 추세, 주요 기여 요인을 자동으로 분석하고 시각화한다.

## 요소
- 데이터 소스(Data Source): 데이터를 가져오는 원본 위치이다.
- 데이터 세트(Data Set): QuickSight에서 쿼리 및 분석을 위해 데이터 소스에서 가져온 데이터의 컬렉션이다.
- 분석(Analysis): 대시보드를 생성하기 위해 데이터를 탐색하고 시각화하는 작업 공간이다.
- 대시보드(Dashboard): 최종 사용자에게 분석 결과를 공유하기 위해 게시된 시각화 보고서이다.

## 특징
- SPICE 엔진: 인메모리 엔진으로, 기본 데이터 소스의 부하를 줄이고 수억 개의 레코드에 대한 실시간 대시보드 성능을 유지한다.
- 서버리스: 인프라 관리나 용량 계획이 필요 없으며, 필요에 따라 자동으로 확장된다.
- 모바일 지원: 모바일 앱을 통해 언제 어디서든 대시보드를 확인하고 상호작용할 수 있다.
- 유연한 요금제: 사용자 단위 요금제와 세션 기반 요금제(익명 액세스가 필요한 경우)를 모두 지원한다.

## 비교
| 구분 | QuickSight (클라우드 BI) | [[Redshift]] (데이터 웨어하우스) |
| :--- | :--- | :--- |
| 주요 역할 | 분석 결과를 시각화하고 공유 | 대규모 데이터를 저장하고 복잡한 쿼리를 실행하여 분석 |
| 데이터 처리 | SPICE 엔진을 통한 인메모리 분석 및 계산 | 컬럼 기반 저장 및 MPP를 통한 쿼리 처리 |
| 관계 | QuickSight는 Redshift를 가장 일반적인 데이터 소스 중 하나로 사용한다. | QuickSight는 분석에 필요한 최종 결과를 Redshift로부터 공급받는다. |

## 예시
- 월간 판매 실적 대시보드:
    1. 마케팅 데이터, 재고 데이터, 판매 트랜잭션 데이터가 [[Redshift]]에 통합되어 저장된다.
    2. 분석가가 QuickSight를 [[Redshift]]에 연결하여 데이터 세트를 생성하고 SPICE 엔진에 데이터를 로드한다.
    3. QuickSight에서 사용자 지정 차트와 그래프를 만들어 실시간 판매 추이, 지역별 수익 등을 분석한다.
    4. 분석이 완료된 대시보드를 경영진과 영업팀에 공유하여 비즈니스 의사 결정을 지원한다.
