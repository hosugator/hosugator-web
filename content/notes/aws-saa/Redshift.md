# Amazon Redshift
## 정의
클라우드에서 페타바이트 규모의 데이터를 분석할 수 있도록 지원하는 빠르고 완전 관리형 데이터 웨어하우징 서비스
## 원리
### 컬럼 기반 저장(Columnar Storage)
- 데이터를 행(Row) 단위가 아닌 열(Column) 단위로 저장하여 데이터 압축률을 높이고, 분석 쿼리 시 필요한 열만 읽어 I/O를 최소화한다.
- 이는 대규모 집계 및 분석 쿼리 성능을 극대화하는 핵심 원리이다.
### MPP (Massively Parallel Processing)
- 클러스터 내의 여러 노드(Node)들이 작업을 병렬로 처리하여 복잡한 쿼리를 빠르게 실행한다.
- 클러스터는 리더 노드(Leader Node)와 컴퓨팅 노드(Compute Node)로 구성된다.
## 요소
- 클러스터(Cluster): 하나 이상의 컴퓨팅 노드와 하나의 리더 노드로 구성된 Redshift의 기본 배포 단위이다.
- 리더 노드(Leader Node): 클라이언트 애플리케이션으로부터 쿼리를 수신하고, 쿼리 계획을 수립한 후 컴퓨팅 노드에 작업을 할당한다.
- 컴퓨팅 노드(Compute Node): 쿼리를 병렬로 실행하고 데이터를 저장하며, 데이터를 노드 간에 분할(Distribution)하여 관리한다.
- Redshift Spectrum: [[S3]]에 저장된 데이터를 Redshift 클러스터에 로드하지 않고 직접 쿼리할 수 있도록 하는 기능이다.
## 특징
- 데이터 압축: 컬럼 기반 저장 방식으로 인해 일반적인 DB보다 높은 압축률을 가지며, 이는 저장 공간 효율성을 높이고 쿼리 속도를 향상시킨다.
- AWS 통합: [[S3]], [[Amazon Kinesis]], [[Glue]] 등과 쉽게 통합되어 ETL(Extract, Transform, Load) 파이프라인 구축이 용이하다.
- 확장성: 노드 추가 및 삭제를 통해 스토리지와 컴퓨팅 용량을 쉽게 확장할 수 있다.
## 비교
| 구분 | Redshift (데이터 웨어하우스) | [[Amazon Relational Database Service (RDS)]] (OLTP) |
| :--- | :--- | :--- |
| 주요 목적 | 대규모 데이터의 분석 및 보고 (OLAP) | 온라인 트랜잭션 처리 (OLTP) |
| 저장 방식 | 컬럼 기반 저장, MPP 아키텍처 | 행(Row) 기반 저장, 단일 서버 아키텍처 |
| 쿼리 복잡성 | 복잡하고 장기 실행되는 집계 쿼리에 최적화 | 짧고 빠른 트랜잭션 쿼리에 최적화 |
## 예시
- 월간 비즈니스 인텔리전스 보고:
    1. 웹 서비스의 운영 데이터가 주기적으로 [[S3]]에 저장되고, [[Kinesis Data Firehose]] 또는 [[Glue]]를 통해 Redshift 클러스터로 로드된다.
    2. 데이터 분석가가 Redshift에서 수천억 건의 판매 기록을 대상으로 월별 사용자 구매 패턴, 지역별 수익 등 복잡한 집계 쿼리를 실행한다.
    3. 컬럼 기반 저장과 MPP 아키텍처 덕분에 이러한 대규모 분석 쿼리가 몇 분 만에 완료되어, 신속한 비즈니스 의사 결정에 필요한 보고서를 생성한다.
