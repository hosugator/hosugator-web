# Snow Family
## 정의
네트워크 대역폭이 제한적이거나 불가능한 환경에서 페타바이트 규모의 대용량 데이터를 [[S3]]로 물리적으로 안전하게 전송하거나, 엣지 환경에서 컴퓨팅을 수행할 수 있도록 설계된 일련의 견고한 물리적 장치 모음
## 요소
- Snowcone: 가장 작고 가벼운 장치 (8TB~14TB). 협소한 공간에서의 데이터 수집 및 엣지 컴퓨팅에 적합하다.
- Snowball Edge: 중간 규모의 장치 (42TB 또는 80TB). 스토리지 최적화와 컴퓨팅 최적화 옵션이 있으며, [[EC2]] 인스턴스를 실행하여 엣지 컴퓨팅을 지원한다.
- Snowmobile: 엑사바이트 규모의 데이터를 AWS로 옮길 때 사용되는 대형 트럭 형태의 컨테이너.
## 원리
### Offline Transfer (오프라인 전송 원리)
대규모 데이터 셋을 인터넷을 통해 전송하는 것이 비현실적일 때 사용된다.
1. 고객이 AWS에 장치를 요청하면, AWS는 장치에 [[KMS]]로 관리되는 암호화 키를 미리 로드하여 배송한다.
2. 고객은 장치를 온프레미스 네트워크에 연결하고, 데이터를 장치에 복사한다.
3. 고객은 장치를 AWS로 반송하고, AWS는 데이터를 장치에서 고객의 [[S3]] 버킷으로 업로드한다.
이 과정에서 데이터 전송 시간과 네트워크 비용을 획기적으로 절감한다.
### Edge Computing (엣지 컴퓨팅)
[[Snowball Edge]]와 [[Snowcone]]은 장치 내에 [[EC2]] 및 [[Lambda]] 기능을 지원하여, 데이터가 생성되는 오프라인 엣지 환경에서 데이터 처리·분석·변환을 수행할 수 있도록 한다.
## 특징
### Security (보안)
데이터는 장치에 복사되기 전후로 자동 암호화(256-bit encryption)되며, 엔드 투 엔드 추적 및 견고한 물리적 보안 기능을 갖추고 있어 전송 중 데이터 보호를 보장한다.
### Cost-Effectiveness (비용 효율성)
네트워크 대역폭이 낮거나, 온라인 전송에 수개월이 소요될 것으로 예상되는 수십 테라바이트 이상의 대용량 데이터를 전송할 때, 온라인 전송 방식(예: [[Direct Connect]] 사용)보다 훨씬 저렴하고 빠르다.
## 예시
### 원격 유전 지대의 대용량 센서 데이터 이관
1. 문제: 인터넷 연결이 불안정한 원격지에 위치한 유전에서 매일 10TB의 센서 데이터를 6개월 동안 AWS로 전송해야 한다. (총 1.8PB)
2. 솔루션: [[Snowmobile]]을 사용하여 데이터를 수집하고 AWS로 물리적으로 운송한다.
3. 결과: 저속 네트워크를 통한 수년에 걸친 전송 대신, 몇 주 내에 안전하게 데이터 마이그레이션을 완료하여 분석 시스템에 빠르게 데이터를 공급한다.
