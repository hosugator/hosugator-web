# 수직적 크기 조정 (Vertical Scaling)
## 정의
단일 서버 또는 인스턴스의 컴퓨팅 리소스를 늘려 처리 능력을 확장하는 방식.

## 요소
### 1. 단일 인스턴스
성능 확장의 대상이 되는 물리적 또는 가상 서버이다.

### 2. 컴퓨팅 리소스
- CPU 및 RAM: 인스턴스에 할당되는 핵심 리소스로, 이를 증설하여 성능을 높인다.
- 스토리지: 볼륨 크기 및 [[IOPS]]를 증가시켜 I/O 성능을 향상시킨다.

## 원리
### 성능 업그레이드 메커니즘
애플리케이션에 대한 수요가 증가하면, 현재 실행 중인 단일 인스턴스를 일시적으로 중단한다.
중단 후 해당 인스턴스의 인스턴스 유형을 더 높은 사양(더 많은 vCPU, 메모리)으로 변경하여 성능을 업그레이드한다.
이러한 작업은 시스템의 기술적 상한선에 도달할 때까지 반복적으로 수행된다.

### 가동 중단 필요성
일반적으로 인스턴스 유형을 변경하거나 리소스를 추가할 때 시스템의 일시적인 가동 중단(Downtime)이 필요하다.

## 특징
- 단순성: 아키텍처 변경 없이 단일 구성 요소만 관리하므로 구현이 간단하다.
- 기술적 한계: 각 인스턴스 유형이나 물리적 서버에는 최대 용량의 한계가 존재한다.
- 단일 장애 지점 (SPOF): 모든 로드가 단일 인스턴스에 집중되어, 해당 인스턴스에 장애가 발생하면 전체 서비스가 중단된다.
- 비용 최적화 (단점): 특정 시점 이후에는 더 큰 인스턴스로의 전환 비용이 매우 가파르게 증가하여 효율성이 떨어진다.

## 비교
| 구분 | 수직적 크기 조정 (Vertical Scaling) | 수평적 크기 조정 (Horizontal Scaling) |
| :--- | :--- | :--- |
| 방법 | 단일 인스턴스의 용량 증설 | 여러 인스턴스의 개수 증설 |
| 복원력 | 낮음 (단일 장애 지점) | 높음 (로드 분산을 통한 내결함성) |
| 탄력성 | 낮음 (수동 작업, 가동 중단 필요) | 높음 (자동 크기 조정을 통한 탄력성) |
| 적합 워크로드 | 라이선스 제약이 있는 애플리케이션, 데이터베이스 (작은 규모) | 웹/앱 서버, API 게이트웨이 (대규모 분산 처리) |

## 예시
### EC2 인스턴스 사양 업그레이드
1. 운영 중인 AWS EC2 `t3.medium` 인스턴스의 CPU 사용률이 지속적으로 90%를 초과하는 성능 병목 현상을 확인한다.
2. 장기적인 확장보다는 단기적인 성능 개선이 시급하다고 판단하여 수직적 크기 조정을 선택한다.
3. EC2 콘솔 또는 CLI를 사용하여 해당 인스턴스를 중지(Stop)한다.
4. 인스턴스 설정을 수정하여 인스턴스 유형을 `t3.medium`에서 `m6i.large`와 같은 더 높은 사양으로 변경한다.
5. 인스턴스를 재시작(Start)하여 더 높은 사양으로 서비스를 재개한다.
