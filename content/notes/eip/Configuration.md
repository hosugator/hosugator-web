# Configuration (형상)
## 정의
[[SDLC]]의 특정 시점에서 정의된 소프트웨어 제품의 기능적/물리적 특성의 집합
## 형상 관리(Configuration Management)
SDLC 도중 소프트웨어 제품의 무결성/안정성을 보장하기 위한 일련의 체계적인 활동
### 형상 항목(Configuration Item, CI)
형상 관리가 필요한 최소 단위(예: 특정 버전의 A 모듈 소스 파일, B 기능 명세 문서)
### 형상 식별(Identification)
관리 대상을 정의한다.
소프트웨어 구성 요소 중 버전 관리가 필요한 요소들(소스 코드, 설계 문서, 테스트 케이스 등)을 명확하게 구분한다.
최초 정의를 [[Base Line]]이라고 부르며, 이후로는 형상 통제 절차를 통해서만 변경할 수 있다.
### 형상 통제(Control)
[[CCB]]의 심의를 거쳐 형상 항목의 변경을 통제한다.
### 형상 감사(Auditing)
베이스 라인의 무결성 검사
형상 기록의 정확성 검증
소프트웨어 제품의 기능적/물리적 감사
- 기능적 감사: 요구사항 합치성
- 물리적 감사: 프로토콜 합치성
### 형상 기록(Status Accounting)
형상 관리 활동 전반에 대한 수행 결과의 기록/보고
소프트웨어의 현상태, 변경이력, 통제결과 등을 추적하고 문서화하여 투명성 확보
