# 데이터 베이스 설계 Data Base Design
## 정의
추상적인 사용자 요구사항을 구체적인 운영 시스템으로 변환하는 계층적 과정
## 종류
### 요구사항 분석
데이터베이스를 사용하는 용도 파악 ([[Requirements]])
### 개념적 설계 (추상적 모델링)
개념적 설계는 어떤 DBMS를 사용할지 고려하지 않고 현실 세계의 데이터를 모델링한다.
역할: 데이터의 의미적 구조를 파악하며, 결과물로 E-R 다이어그램이 나온다.
### 논리적 설계 (DBMS 종속적 구조화)
논리적 설계는 개념적 설계 결과를 목표 DBMS가 이해할 수 있는 테이블 구조로 변환하는 단계이다.
작업: E-R 다이어그램을 테이블 형태로 변환하고, 정규화를 수행하여 데이터의 중복 및 [[Anomaly]](갱신/삽입/삭제 이상)을 제거한다.
차이점: DBMS의 종류에는 종속되지만, 특정 DBMS의 물리적 특성에는 독립적이다.
### 물리적 설계 (성능 최적화 정의)
물리적 설계는 특정 DBMS의 특성을 고려하여 성능 최적화에 중점을 두고 모든 상세 스펙을 정의하는 단계이다.
작업 내용: 인덱스 유형/위치, 저장 공간 할당, 데이터 분할(파티셔닝) 방법 등을 결정한다.
결과물: 테이블 정의서, 인덱스 명세서와 같이 구현에 필요한 상세 지침(문서)을 완성한다.
물리적 설계에서는 실제 SQL 코드를 작성하거나 실행하지 않는다. 다만, 코드가 어떻게 작성되어야 하는지에 대한 규칙(아키텍처/표준)을 명세한다.
### 구현 (실제 DB 객체 생성)
구현 단계는 물리적 설계에서 정의된 모든 명세서를 바탕으로 실제 코드를 작성하고 실행하는 단계이다.
작업 내용: `CREATE TABLE`, `CREATE INDEX` 등의 SQL DDL 문을 작성하고 DBMS에 실행하여 DB 객체를 물리적으로 생성한다.
