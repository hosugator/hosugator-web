## 3. Technical Experience (엔지니어링 경력 & 프로젝트)
> 작성 공식 (STAR-L): Situation(문제) -> Task(과제) -> Action(내가 한 기술적/전략적 결정) -> Result(수치적 성과)
### [프로젝트/경력 명] (기간)
한 줄 요약: (예: MSA 아키텍처 도입을 통한 대규모 트래픽 처리 안정성 확보)
- Problem: (예: 모놀리식 구조에서 AI 모델 추론 시 전체 서버 응답 지연 발생)
- Solution:
    - FastAPI의 비동기(Async) 처리를 도입하여 I/O 블로킹 해결
    - MSA 아키텍처로 전환하여 모델 서버와 백엔드 서버의 부하 격리
    - 4-bit Quantization 적용으로 메모리 사용량 50% 절감
- Impact: 응답 속도 60% 단축 및 AWS 비용 30% 절감
## 4. Professional Experience (PM 경력)

### [회사명] (기간) / Project Manager

- Communication: 비개발 직군(고객)의 모호한 요구사항을 기술 명세(Spec)로 구체화하여 엔지니어링 리소스 낭비 방지
- Problem Solving: 공기 지연 리스크 상황에서 공정 최적화 아이디어를 제안하여 Q/C/D(품질/비용/납기) 목표 달성
