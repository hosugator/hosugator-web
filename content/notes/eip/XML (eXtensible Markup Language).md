# XML의 개념 및 특징
XML은 태그를 사용하여 데이터를 표현하며, 데이터를 계층적으로 구조화하고 사용자 정의 태그를 통해 데이터의 의미(Semantics)를 부여하는 데 중점을 둔다.
## 개발 배경
1. SGML의 복잡성 해결: SGML의 장점인 확장성은 취하면서 복잡성을 줄여 경량화되었다.
2. HTML의 한계 극복: HTML이 데이터의 형식만 정의하는 것과 달리, XML은 사용자가 태그를 정의하여 데이터의 구조와 의미를 부여하는 확장성(eXtensible)을 제공한다.
## 키-값 쌍과 XML
XML은 태그(키)와 태그 사이의 내용(값)을 사용하여 데이터를 표현하므로 넓은 의미에서 키-값 쌍을 가진다. 핵심은 단순히 쌍을 나열하는 것이 아니라 태그 중첩을 통해 복잡한 트리 구조를 표현하는 것이다.
# XML, JSON, Dictionary의 차이
Dictionary는 메모리 내부의 자료 구조로 XML/JSON과 계층 및 목적이 다르다. XML과 JSON은 프로그램 간 데이터 교환을 위한 외부 포맷이다.
## 목적과 활용
Dictionary: 프로그램이 실시간으로 데이터를 저장하고 처리하기 위한 내부 자료 구조이다.
XML/JSON: 서로 다른 시스템 간에 데이터를 교환하기 위한 표준화된 포맷으로, API 통신이나 파일 저장에 사용된다.
## XML과 JSON의 상호 대체성
대부분의 상황에서 상호 대체 가능하지만, 완벽하지는 않다.
XML은 복잡한 문서 구조 및 DTD, XML Schema를 통한 엄격한 유효성 검사가 필요할 때 선호된다.
JSON은 파일 크기가 작고 파싱 속도가 빨라, 웹 서비스처럼 빈번하고 가벼운 데이터 교환에 주로 사용된다.
