## 구조적 방법론의 높은 종속성이라는 한계
구조적 방법론(예: C 언어의 함수)에서 기능과 데이터가 분리된다는 것은 논리적 관리 주체가 다르다는 것을 의미한다.
* 기능은 외부 데이터에 종속적이다. 기능(함수)은 독립적으로 존재하며, 데이터를 외부로부터 파라미터로 전달받아 처리한다.
* 데이터 구조는 기능 외부에 정의된 전역 변수 또는 구조체 형태로 존재한다.
* 문제점: 외부 데이터의 형태나 구조가 변경될 경우, 그 데이터를 사용하는 모든 기능(함수)을 일일이 수정해야 한다. 이는 시스템의 높은 결합도로 이어져 유지보수를 어렵게 만드는 요인이다.
## 객체지향 방법론의 독립성 확보
객체지향 방법론은 캡슐화를 통해 데이터와 기능을 통합하여 외부 변경에 대한 독립성을 확보한다.
* 통합의 의미: 데이터(속성)와 기능(메소드)은 클래스라는 하나의 논리적 캡슐 안에 통합된다.
* 작동 원리: 기능은 외부에서 데이터를 전달받지 않고, 자신이 소유한 상태(인스턴스 변수)를 직접 변경하거나 조회한다.
* 독립성 확보: 외부로부터 데이터를 가져와 클래스 내부의 고유한 상태로 저장하고 그 상태를 중심으로 기능을 수행한다.
* 이 상태(데이터)는 클래스 외부에 직접 노출되지 않도록 정보 은닉되어, 데이터의 내부 구현 방식이 바뀌더라도 외부에 정의된 인터페이스(기능 호출 방식)는 유지될 수 있다.
* 이는 시스템의 낮은 결합도와 높은 응집도를 달성하여 재사용성과 유지보수성을 높이는 핵심이다.
