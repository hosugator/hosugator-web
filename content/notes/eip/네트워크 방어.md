# Network Defense
## 핵심 용어
| 용어 | 정의 |
| :--- | :--- |
| Land Attack | 공격자가 패킷의 출발지/목적지 주소를 동일하게 하여 대상 컴퓨터의 실행속도를 느리게 하거나 마비시키는 서비스 거부 공격이다. |
| DoS | Denial of Service. 표적이 되는 서비스 자원을 고갈시킬 목적으로 단일 공격자가 데이터를 집중적으로 전송하는 공격이다. |
| DDoS | Distributed Denial of Service. 다수의 좀비 PC(Bot)를 이용해 분산된 공격을 수행하는 방법이다. |
| IP | 네트워크 주소를 기반으로 패킷을 라우팅하는 네트워크 계층 프로토콜이다. |
| ICMP | IP 통신 중 발생하는 네트워크 오류를 알리거나 진단하는 데 사용되는 프로토콜이다. |
| 3-Way Handshake | TCP 통신에서 클라이언트와 서버 간에 연결을 확립하기 위한 3단계 동기화 과정이다. |
## 정의
사이버 공격, 데이터 유출, 무단 접근 등 다양한 위협으로부터 컴퓨터 네트워크를 보호하는 활동과 전략 일체
## 원리
### SYN Flooding 방어 (제어 규칙의 적용)
취약점: 서버가 3단계 ACK 없이 연결 대기 상태를 유지하며 메모리 자원이 고갈된다.
방어 메커니즘 (제어 규칙):
1.  SYN Cookie: 서버가 메모리에 상태를 저장하지 않고 쿠키를 사용하여 자원 소모 방지.
2.  타임아웃 단축: `SYN-ACK`에 대한 응답 대기 시간을 짧게 설정하여 '일정 시간 응답하지 않으면 종료'하는 규칙을 엄격하게 적용.
### Teardrop 방어 (오프셋 유효성 검사)
취약점: 조작된 오프셋 정보로 인해 수신 시스템의 재조립 로직이 오류에 빠지거나 자원을 과도하게 사용한다.
오프셋 (Offset): 분할된 패킷(단편)이 원본 데이터의 어느 위치부터 시작하는지를 알려주는 정보이다.
방어 메커니즘 (제어 규칙):
1.  오프셋 유효성 검사: 재조립 전, 단편들의 오프셋 정보가 논리적으로 모순되지 않는지 엄격하게 검사.
2.  부적합 패킷 즉시 폐기: 유효성 검사에 실패한 조작된 단편들은 재조립 없이 즉시 폐기하여 자원 소모를 원천 차단.
## 현대적인 DoS/DDoS 방어 전략 (지능형 방어)
핵심 문제: 정상적인 요청처럼 보이는 대규모 트래픽(Layer 7 공격)을 구별하여 방어하는 것이다.
### 용량 기반 방어 (Volumetric Defense - Layer 3/4)
-   클라우드 기반 스크러빙 센터: 대규모 공격 트래픽을 대용량 네트워크로 우회시켜 필터링하고 깨끗한 트래픽만 서버로 전송하여 용량 자체를 흡수한다.
-   CDN 및 분산 로드 밸런싱: 공격 트래픽을 단일 서버가 아닌 분산 노드로 나누어 공격 효과를 희석시킨다.
### 지능형 행위 분석 기반 방어 (Behavioral Analysis - Layer 7)
-   율 제한 (Rate Limiting): 특정 소스의 요청 속도를 정상 임계치로 제한하여 비정상적인 요청 폭주를 차단한다.
-   행위 기반 분석: 머신러닝을 통해 정상 사용자의 패턴을 학습하고, 공격 봇의 비인간적인 패턴(일정한 요청 간격, 불일치 헤더 등)을 식별하여 차단한다.
-   도전(Challenge) 기법: CAPTCHA나 JavaScript 연산을 요구하여 요청자가 공격용 봇이 아닌 인간임을 능동적으로 증명하도록 요구한다.
-   지연 및 연결 끊기: 활동이 없는 연결(Slowloris 공격)을 즉시 종료하는 정책을 적용하여 서버의 동시 연결 자원 고갈을 방지한다.
