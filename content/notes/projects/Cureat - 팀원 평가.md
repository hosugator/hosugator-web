### 개인 또는 우리 팀이 잘한 부분과 아쉬운 점
기획 단계에서 구현하려고 했던 주요 기능을 절반 정도 구축했다. Gemini API와 더불어 Naver Service API, Kakao Search API를 사용을 했는데 API 연동이 생각보다 만만치 않았다. 또한 원래 구축하려면 PostgreSQL, PGvector가 아닌 SQlite로 구축을 하게 되어 아쉬움이 많이 남았다. 개인적으로 백엔드 경험이 없는 상태에서 공식문서, 논문, 예제 등을 참고하다 보니 머리가 더 복잡해지는 듯한 기분이 들었고, 개인 역량이 아직 많이 부족하다는 것을 다시 한 번 느꼈다. 역할 분배, 본인의 파트가 아니더라도 서로 도와주며 프로젝트를 진행했던 점이 좋았다. 다른 팀에 비해 인원이 부족했기 때문에 타이트했고, 전문적으로 코드를 잘 아는 인원이 없어서 많이 헤맸던 것을 생각하면 많이 힘들었지만 많이 성장하는 계기가 되기도 했다.
### 프로젝트 결과물의 추후 개선점이나 보완할 점 등 내용 정리 
API 연동, 크롤링, 회원가입, 로그인 기능까지는 프로젝트 이후에 구현이 되었고, 백엔드와 서버를 통해 크롤링으로 데이터를 조금 더 쌓은 후 전체적으로 코드를 수정하고, Postgre, PGvector를 연동할 예정이다. 초창기에 구상했듯이 사용자 정보와 음식점의 일부 메타데이터는 Postgre에 저장하고, 음식점 정보들은 벡터화하여 PGvector에 저장을 하도록 할 예정이다. 또한 기획 때 주요 기능으로 뽑았던 코스추천, 리뷰, 길찾기, 커뮤니티 기능들도 전부 구현을 해서 하나의 앱으로 만들어보고 싶다.
### 프로젝트를 수행하면서 느낀 점이나 경험한 성과(경력 계획 등과 연관) 
협업을 해본 경험이 많이 않아서 아직 갈 길이 멀다는 것을 느꼈다. 특히 프론트와 백엔드를 연결하는 과정에서 서로 엔드포인트를 맞추지 않았거나, 백엔드의 서버 설정 자체가 잘못되어 생긴 간단하고 사소한 문제 때문에 많은 시간을 허비했다는 점이 아쉬웠다. 나름 꼼꼼하게 본다고 봤지만 사소한 실수도 있었고, 코드 자체를 뜯어 고치는 경험도 해보았다. Git Hub와 Jira를 적극적으로 써본 것은 앞으로의 단계에도 많은 도움이 될 것 같다. 힘들었지만 좋은 경험이었고 다음 프로젝트는 어떻게 나아가야 하는지에 대한 방향을 잡을 수 있게 되었다.
