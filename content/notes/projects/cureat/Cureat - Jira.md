## 1단계: 백로그(Backlog)에 작업 쌓기
백로그는 프로젝트에 필요한 모든 작업 목록을 모아두는 곳입니다. 스프린트를 시작하기 전에, 기획서에 있는 모든 기능을 Jira의 '이슈(Issue)'로 만듭니다.
### Jira 이슈 종류 선택:
 - Epic: 가장 큰 단위의 기능(예: `사용자 인증 시스템`, `맛집 검색 기능`, `추천 엔진 개발`). 이 에픽을 중심으로 하위 작업을 묶어 포트폴리오에 큰 틀을 보여주기 좋습니다.
 - Story: 사용자의 관점에서 정의된 기능(예: `사용자는 로그인을 할 수 있다`, `사용자는 맛집을 검색할 수 있다`).
 - Task: 개발자가 수행해야 할 구체적인 작업(예: `회원가입 API 개발`, `로그인 UI 구현`, `BERT 모델 학습`).
### 작업 분해하기:
 - 기획서의 모든 핵심 기능을 Story로 만듭니다.
 - 각 Story를 개발자별로 Task 단위로 쪼갭니다. 예를 들어, `로그인`이라는 Story 아래에 `백엔드: 로그인 API 개발`, `프론트엔드: 로그인 UI 개발`과 같은 Task를 만듭니다.
### 담당자 지정하기:
- 각 Task에 담당자를 지정하여 책임 소재를 명확히 합니다.
## 2단계: 스프린트(Sprint) 운영하기
- 스프린트 만들기: Jira에서 8일짜리 스프린트를 생성합니다.
- 작업 할당: 백로그에서 이번 스프린트에서 할당된 Task들을 드래그해서 스프린트 보드로 옮깁니다. 모든 팀원이 각자의 역할에 맞는 작업을 보드에 추가합니다.
- 일일 스탠드업: 매일 아침 Discord에서 논의한 내용을 바탕으로 각자의 작업 상태를 'To Do' -> 'In Progress' -> 'Done'으로 옮깁니다. 이는 오프라인 협업의 실시간 진행 상황을 온라인에 기록하는 중요한 과정입니다.
## 3단계: 보드(Board)로 진행 상황 시각화하기
Jira 보드는 프로젝트의 진행 상황을 한눈에 볼 수 있는 시각화 도구입니다.
- 칸반(Kanban) 보드: `To Do`, `In Progress`, `Done` 세 가지 열(column)을 만들어 현재 진행 중인 모든 작업을 시각적으로 보여줍니다.
- 작업 상태 업데이트: 개발자가 작업을 완료하면 해당 Task 카드를 'Done'으로 옮깁니다. PM은 이 보드를 통해 전체 프로젝트의 진척도를 쉽게 파악할 수 있습니다.
## 포트폴리오를 위한 활용 팁
- 이슈 설명 상세화: 각 이슈에 작업 목표와 기술 스택, 예상 소요 시간 등을 자세히 기록하면 포트폴리오에 '계획성'과 '문제 해결 과정'을 보여주는 자료로 활용할 수 있습니다.
- 자동화 연동: GitHub와 Jira를 연동하면, 특정 이슈 번호를 포함한 커밋(Commit)이 자동으로 Jira 이슈에 기록됩니다. 이는 작업 이력을 더욱 풍부하게 만들어 줍니다.
- 보고서 활용: Jira의 'Reports' 기능을 이용하면 팀원별 작업량, 프로젝트 완료율 등 다양한 통계 데이터를 자동으로 생성해 줍니다. 이 데이터를 포트폴리오에 첨부하면 프로젝트 성과를 정량적으로 증명할 수 있습니다.
이 3단계 프로세스를 따라가면, 단순히 '작업을 했다'는 것을 넘어 '어떤 목표를 어떻게 달성했는지'를 체계적으로 기록할 수 있으며, 이는 여러분의 포트폴리오를 더욱 빛나게 할 것입니다.
