# 기획 멘토링 (박수연 멘토)
## 비즈니스 모델(BM)
- 광고 외 대안 모색: 광고가 가장 쉬운 BM이지만, 기획 의도에 맞지 않다면 다른 모델을 고려해야 합니다.
- 월 구독 시스템: 단순한 리포트 제공을 넘어, 신규 고객을 유입시키는 월 구독 시스템을 구축해야 합니다.
- B2B 데이터 판매: 사용자 데이터가 충분히 쌓이면, 익명화된 지역별 상권 분석 데이터나 트렌드 데이터를 대형 프랜차이즈, 편의점, 식자재 업체 등에 판매할 수 있습니다.
- 커뮤니티 기반 마켓플레이스: '미식가'나 '맛잘알' 같은 특정 사용자를 끌어들여 커뮤니티를 형성하고, 이들이 제공하는 고급 정보를 유료 구독자에게 제공하는 방식으로 차별성을 확보할 수 있습니다.
- B2B 제휴: 요식업체, 전시회, 공연, 교통수단 등과 제휴하여 할인 쿠폰을 제공함으로써 소비자에게 혜택을 주고, 점주에게는 마케팅 효과를 줍니다.
## 평가 지표
- 사용자 경험(UX) 지표: 기술적 지표보다는 사용자의 만족도에 집중해야 합니다.
 - 광고 제거에 따른 사용자 만족도
 - 검색 시간 단축률
 - 고객 기반 사용자 만족도 조사
- 리뷰와 데이터 확보: 재방문 의사에 대한 여러 유형의 질문을 포함하고, 리뷰 이벤트를 통해 참여율을 높이며, 직접 리뷰 작성 검증 방법을 마련해야 합니다.
# 개발 멘토링 (최재흥 멘토)
## 개발 운영 지표
- 성과 지표:
 - 버그 발생률 및 기능 구현 성공률
 - 애자일 스토리 포인트 달성률 (예: 100점 중 달성률)
- 안정성 및 성능:
 - 서버 비용 감축 및 서버 퍼포먼스(동작 시간) 계산
 - API 비용 절감
- 데이터 관리:
 - 사용자 요청 데이터를 바로 DB에 저장하기보다는, 먼저 파일로 저장한 후 해당 파일의 URL을 생성하여 DB에 저장하는 방식을 사용하면 효율적입니다.
## 협업 및 업무 분담
- 프론트엔드 업무 분담: 일반적으로 화면 단위로 폴더를 나누어 관리하고 작업합니다. 비슷한 페이지는 조건문을 사용하여 코드를 재활용할 수 있습니다.
- 명세서 작성: 프론트엔드와 백엔드가 함께 명세서를 협의하여 작성하는 것이 중요합니다.
- Flutter 추천: 모바일 앱 개발 시, 화면 단위로 폴더를 나누고 섹터별로 업무를 분담하는 Flutter를 추천합니다.
