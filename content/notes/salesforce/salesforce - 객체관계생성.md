# 객체 관계 생성 (Create Object Relationships)

## 관계의 정의 및 역할

객체 관계 (Object Relationships)는 두 객체를 연결하는 **특수한 필드 유형**입니다. 이 관계는 영업 사원이 거래처 (Account) 레코드에서 해당 연락처 (Contact)를 쉽게 조회할 수 있도록 표준으로 제공됩니다. 관계를 설정함으로써 서로 다른 객체의 정보를 **Related Tab**에서 통합적으로 확인할 수 있습니다.

### 관계 필드의 기능

관계 필드는 단순한 데이터 연결을 넘어 객체 간의 **연결 고리**를 만듭니다. 이는 표준 관계 (Standard Relationship, 예: Account to Contact)뿐만 아니라 사용자의 비즈니스 요구에 따른 **사용자 정의 관계 (Custom Relationship)** 구축을 가능하게 합니다.

## 주요 관계 유형

Salesforce에서 사용자 정의로 생성할 수 있는 주요 관계 유형은 크게 두 가지입니다.

### 조회 관계 (Lookup Relationship)

조회 관계는 두 객체를 연결하는 가장 일반적인 유형입니다.

|특징|설명|
|---|---|
|연결 강도|느슨한 연결 (Casual)|
|데이터 종속성|자식 객체가 부모 객체 없이도 **독립적으로 존재**할 수 있습니다.|
|삭제 행동|부모 레코드 (예: Account)가 삭제되어도 자식 레코드 (예: Contact)는 **삭제되지 않고 유지**됩니다.|
|사용 시점|객체들이 **일부 경우에만** 관련되거나, 자식 객체가 독립적으로 기능해야 할 때 사용됩니다.|
|카디널리티|일대일 (One-to-One) 또는 **일대다 (One-to-Many)** 관계가 가능합니다.|

### 마스터-디테일 관계 (Master-Detail Relationship)

마스터-디테일 관계는 부모 객체가 자식 객체의 특정 동작을 제어하는 강력한 연결입니다.

|특징|설명|
|---|---|
|연결 강도|강력한 연결 (Tighter)|
|데이터 종속성|디테일 (자식) 객체 레코드는 **마스터 (부모) 객체에 종속**됩니다. 마스터 없이는 존재할 수 없습니다.|
|삭제 행동|마스터 레코드가 삭제되면 모든 관련 디테일 레코드도 **자동으로 삭제**됩니다 (예: Property 삭제 시 Offer도 삭제).|
|제어 기능|마스터 객체가 디테일 객체의 **보안 및 접근 권한**을 제어합니다.|
|필드 생성 위치|관계 필드는 **디테일 (자식) 객체**에 생성됩니다.|

## 기타 관계 및 유의사항

### 계층 관계 (Hierarchical Relationship)

계층 관계는 조회 관계의 **특수한 형태**입니다. 일반적인 조회 관계와 달리 **User 객체**에만 사용할 수 있으며, 주로 사용자 간의 **관리 체계 (Management Chains)**나 상위/하위 사용자 연결을 구축하는 데 사용됩니다.

### 데이터 모델 복잡성 관리

객체 간에 관계를 추가할수록 데이터 모델의 복잡성은 증가합니다. 따라서 객체, 레코드 또는 필드를 변경하거나 삭제할 때는 **관계 동작 (Relationship Behaviors)**을 면밀히 확인하는 등 **추가적인 주의**가 필요합니다. 관계가 설정되면 데이터 무결성과 시스템 연동에 영향을 미치기 때문입니다.
