# 데이터 보안 개요 (Overview of Data Security)
## 데이터 접근 제어의 중요성
Salesforce 조직(org)의 데이터 보안은 **적절한 사용자에게 적절한 데이터 접근 권한**을 부여하는 데 중점을 둡니다. 이는 기밀 데이터(예: 사회 보장 번호)를 보호하면서도, 모든 사용자가 자신의 업무를 수행하는 데 필요한 데이터에 **쉽게 접근**하도록 균형을 맞추는 것입니다. Salesforce의 보안 모델은 유연하고 **계층화**되어 있어, API 수준에서도 보안이 보호됩니다.
## 데이터 접근을 제어하는 4가지 계층
Salesforce는 네 가지 주요 계층에서 데이터 접근을 제어하며, 가장 넓은 범위부터 가장 좁은 범위까지 순차적으로 적용됩니다.
### 1. Organization (조직)
* **제어 대상:** **전체 조직**에 대한 접근을 관리합니다.
* **주요 기능:** 승인된 사용자 목록 유지, **암호 정책** 설정, 특정 시간 및 위치(IP 주소)에서만 로그인 허용(Login Hours and IP Ranges).
### 2. Objects (객체)
* **제어 대상:** **특정 객체 유형**에 대한 접근을 관리합니다.
* **주요 기능:** 사용자 그룹이 해당 객체의 레코드를 **생성, 보기, 편집, 삭제**할 수 있는지 설정합니다. (**Permission Sets** 사용 권장)
    * *예시:* 면접관은 포지션 레코드를 **볼 수만** 있게 설정.
### 3. Fields (필드)
* **제어 대상:** 객체에 접근 권한이 있더라도 **특정 필드**에 대한 접근을 제한합니다.
* **주요 기능:** 민감한 필드(예: 보너스 금액)를 특정 사용자 그룹에게 **보이지 않게** 설정합니다. (**Permission Sets** 사용 권장)
### 4. Records (레코드)
* **제어 대상:** 객체 및 필드 접근 권한이 있더라도 **개별 객체 레코드**에 대한 접근을 제한합니다.
* *예시:* 면접관이 **자신의 리뷰**는 보지만, **다른 면접관의 리뷰**는 보지 못하도록 설정.
## 레코드 수준의 데이터 접근 관리 (Records Level)
레코드 수준 접근 권한은 네 가지 방식으로 관리됩니다. 보안은 **가장 제한적인 설정**에서 시작하여 점차적으로 권한을 부여합니다.
### 1. Organization-Wide Defaults (조직 전체 기본값)
* **역할:** 데이터 접근의 **기본 수준**을 설정하여, 데이터를 **가장 제한적인 수준으로 잠급니다.** 다른 모든 레코드 보안 도구는 이 기본값 위에 **추가 권한**을 부여하는 데 사용됩니다.
### 2. Role Hierarchies (역할 계층)
* **역할:** 계층 구조상 **상위 사용자**가 자동으로 하위 사용자가 **소유한 모든 레코드**에 접근할 수 있도록 권한을 부여합니다.
### 3. Sharing Rules (공유 규칙)
* **역할:** 조직 전체 기본값에 대한 **자동 예외**를 생성합니다. 특정 그룹의 사용자들이 자신이 소유하지 않은 레코드에 접근할 수 있도록 **추가 권한**을 부여합니다.
### 4. Manual Sharing (수동 공유)
* **역할:** 레코드 소유자가 다른 사용자에게 레코드를 **일시적으로 공유**하도록 허용합니다. (자동화되지 않은 일회성 공유)
## 시스템 사용 감사 (Audit System Use)
감사(Auditing)는 잠재적인 보안 문제 진단 및 대응을 위한 핵심 정보를 제공합니다.

| 감사 도구 | 추적 대상 |
| :--- | :--- |
| **Record Modification Fields** | 레코드를 생성하거나 **마지막으로 수정한 사용자**의 정보를 저장합니다. |
| **Login History** | 지난 6개월간의 **로그인 성공/실패 시도** 기록입니다. |
| **Field History Tracking** | 개별 필드 값의 **변경 사항**을 자동으로 추적합니다. |
| **Setup Audit Trail** | 조직의 **구성 설정(Configuration)**에 대한 변경 사항을 기록합니다. |
| **Event Monitoring** | 조직 내 사용자 활동에 대한 **세부적인 이벤트** 정보를 제공하여 비정상 행동을 식별합니다. |
