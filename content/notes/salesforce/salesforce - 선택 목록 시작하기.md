# 선택 목록 시작하기 (Get Started with Picklists)

## 핵심 용어 정의

### 선택 목록 해부 (Anatomy of a Picklist)
선택 목록 (Picklist)은 사용자가 미리 정의된 옵션 목록에서 값을 선택할 수 있도록 제한하는 필드 유형입니다. 이는 데이터의 일관성을 유지하고 입력 오류를 줄이는 데 사용됩니다.

| 용어 | 정의 | 관리 위치 |
| :--- | :--- | :--- |
| **Field** | 선택 목록의 **유형**과 **속성**을 정의하는 부분입니다. (예: 다중 선택 가능 여부, 종속성 여부) | 필드 수준 (Field Level)에서 관리 |
| **Value Set** | 사용자가 보게 되는 **선택지**, 순서, 기본값 등을 정의하는 부분입니다. | 값 집합 (Value Set) 수준에서 관리 |

## 선택 목록 유형 및 속성

### 선택 목록의 유형
Salesforce에서 제공하는 선택 목록은 기능 및 정의 방식에 따라 세 가지 주요 유형으로 나뉩니다.

| 유형 | 설명 | 값 변경 유연성 |
| :--- | :--- | :--- |
| **Standard Picklists** | Salesforce 조직에 기본적으로 포함된 선택 목록입니다 (예: Lead Source, Opportunity Stage). | 종종 표준 값 집합을 공유하므로, 한 곳에서 값을 변경하면 공유하는 다른 필드에도 반영됩니다. |
| **Custom Picklists** | 사용자가 직접 생성하고 값을 정의하는 선택 목록입니다. | 가장 유연하며, 값과 동작을 직접 구성할 수 있습니다. |
| **Custom Multi-Select Picklists** | 사용자가 목록에서 **하나 또는 그 이상의 값**을 선택할 수 있도록 허용하는 사용자 정의 선택 목록입니다. | 단일 선택 목록보다 값의 총 개수 제한이 낮습니다. |

### 선택 목록의 주요 속성 및 정의 방식

| 속성/정의 방식 | 설명 | 특징 |
| :--- | :--- | :--- |
| **Restricted Picklists** | **제한된 선택 목록**. 관리자가 승인한 값 집합으로 선택 항목을 제한하여, API나 다른 앱을 통한 새 값 추가를 차단합니다. | 데이터 일관성을 유지하고 입력 오류(오타)를 방지하는 데 유용합니다. |
| **Dependent Picklists** | **종속 선택 목록**. 다른 필드(제어 필드)의 선택 값에 따라 표시되는 값 목록이 필터링되는 선택 목록입니다. | UI 공간을 절약하고 사용자에게 안내를 제공하며 데이터 무결성을 향상시킵니다. |
| **Controlling** | 종속 선택 목록의 값을 필터링하는 데 사용되는 다른 필드(선택 목록 또는 체크박스)입니다. | 제어 필드입니다. |
| **Global Value Set** | **전역 값 집합**. 여러 사용자 정의 선택 목록 필드에서 공유하기 위해 생성되는 사용자 정의 값 집합입니다. | 여러 필드에 걸쳐 값 목록을 일관되게 관리합니다. |

## 사용자 정의 선택 목록 생성 단계

### 사용자 정의 단일 선택 목록 생성
**Picklist**를 필드 유형으로 선택하여 사용자 정의 선택 목록을 생성합니다.

1.  Setup에서 **Object Manager** 탭을 클릭하고 선택 목록 필드를 포함할 객체를 선택합니다.
2.  **Fields & Relationships**를 클릭합니다.
3.  **New**를 클릭하고 **Picklist**를 선택한 후 Next를 클릭합니다.
4.  필드에 Label을 입력합니다. Field Name (API 이름)은 자동으로 할당되며, **__c**로 끝납니다.
5.  **Enter values, with each value separated by a new line**를 선택하고 값을 입력합니다.
6.  선택 사항으로 값을 알파벳순으로 정렬하거나 목록의 첫 번째 값을 기본값으로 설정합니다.
7.  **Restrict picklist to the values defined in the value set** 옵션을 선택하여 **제한된 선택 목록**으로 설정할지 여부를 결정합니다.
8.  Next를 클릭하여 필드 수준 보안을 설정합니다.
9.  Next를 클릭하여 선택 목록 필드를 포함할 페이지 레이아웃을 선택합니다.
10. Save를 클릭합니다.

## 다중 선택 목록의 고려 사항

### Custom Multi-Select Picklists 사용 시점
사용자에게 여러 값을 동시에 선택하도록 허용해야 할 때 사용합니다. 선택된 값은 세미콜론(;)으로 구분되어 필드에 표시됩니다.

* **단일 선택 목록 (Picklist)**: 한 번에 하나의 값만 선택할 수 있도록 제한합니다.
* **다중 선택 목록 (Multi-Select Picklist)**: 한 번에 하나 또는 그 이상의 값을 선택할 수 있습니다.

### 다중 선택 목록의 주의사항
| 항목 | 주의 사항 |
| :--- | :--- |
| **값 제한** | 일반 선택 목록보다 총 값의 개수 제한이 더 낮습니다. |
| **필드 유형 변환** | 다중 선택에서 단일 선택으로 변환할 경우, 기존 레코드의 해당 필드 값이 **지워집니다**. |
| **보고서 및 대시보드** | 보고서 작성 시, 단일 값 검색이 아닌 **contains** 또는 **includes** 연산자를 사용해야 다중 선택 결과를 모두 수집할 수 있습니다. |
| **수식 참조** | 특정 함수만 다중 선택 목록을 수식에서 참조할 수 있습니다. |
