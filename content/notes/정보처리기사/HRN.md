# Highest Response Ratio Next
## 정의
대기 시간·서비스 시간을 조합하여 [[SJF]]의 약점인 기아 현상을 보완하는 스케줄링 기법

$\text{우선순위} = \frac{\text{대기 시간} + \text{서비스 시간}}{\text{서비스 시간}}$
## 요소

| 용어                    | 정의                                            |
| :-------------------- | :-------------------------------------------- |
| 대기 시간 (Wait Time)     | 프로세스가 CPU를 할당받기 위해 준비 큐에서 기다린 총 시간이다.         |
| 서비스 시간 (Service Time) | 프로세스가 완료되는 데 필요한 총 실행 시간(CPU Burst Time)이다.   |
| 응답 시간 (Response Time) | 프로세스가 도착한 시점부터 CPU를 할당받아 처음으로 실행된 시점까지의 시간이다. |

## 질의
### 분자에 서비스 시간을 넣는 이유
1.  수학적 기본값 설정: 계산식을 $\text{우선순위} = 1 + \frac{\text{대기 시간}}{\text{서비스 시간}}$로 분해하면, 대기 시간이 0인 프로세스는 우선순위 1부터 시작하게 된다.
2.  응답 비율 측정 및 공정성 확보 (핵심): 분자에 서비스 시간을 포함하여 대기 시간의 절대값이 아닌, 서비스 시간 대비 대기 시간의 비율을 측정한다.
     짧은 작업: 서비스 시간(분모)이 작아 대기 시간이 짧아도 우선순위 값이 급격히 커진다 (SJF 효과 유지).
     긴 작업: 서비스 시간(분모)이 커도 대기 시간이 길어질수록 비율 값이 꾸준히 증가하여, 결국 다른 긴 작업보다 높은 우선순위를 획득하고 실행되어 기아(Starvation)를 방지한다.
