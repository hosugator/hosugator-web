# 핵심 용어 정의표

| 용어        | 정의                                                         |
| :-------- | :--------------------------------------------------------- |
| IPSec     | IP 계층(3계층)의 보안 취약점을 해결하기 위해 개발된 보안 프로토콜 집합이다.              |
| IKE       | 공개된 네트워크를 통해 암호화에 사용할 키(Key)를 안전하게 협상하고 교환하는 키 관리 프로토콜이다.  |
| 공유 비밀 키   | 통신 당사자들이 IKE를 통해 안전하게 생성하고 각자 보관하는, 암호화와 복호화에 사용되는 비밀 키이다. |
| 보안 연관(SA) | 두 통신 당사자 간에 보안 통신을 위해 합의된 규칙의 집합으로, 사용될 키와 알고리즘이 포함된다.     |
# IKE의 역할과 안정성 확보 원리
IKE(Internet Key Exchange)는 암호화 통신에서 가장 중요한 요소인 키를 안전하지 않은 공용 네트워크를 통해 안전하게 생성하고 관리하는 핵심 기능을 수행한다.
 IKE는 암호화 프로토콜(ESP) 자체의 안정성을 높인다기보다, 암호화 통신의 근간이 되는 키의 안전한 관리를 제공하여 시스템의 안정성을 높인다.
 IKE가 없으면 키 교환 과정 자체가 공격에 노출될 위험이 크다.
## 키 공유의 안전성 (Diffie-Hellman 교환)
 IKE는 주로 디피-헬만(Diffie-Hellman) 키 교환 알고리즘을 사용한다.
 통신 당사자들이 공개 정보만 주고받아도, 각자의 개인 정보를 결합하여 네트워크상에서 도청당하지 않은 동일한 공유 비밀 키를 독립적으로 생성할 수 있다.
 이 과정을 통해 키를 평문으로 통신에 같이 보내는 위험을 회피한다.
## 보안 연관(SA) 관리
 IKE는 통신 당사자 간에 암호화 알고리즘, 인증 알고리즘, 그리고 사용될 키를 포함하는 보안 연관(SA)을 자동으로 수립하고 관리한다.
 IKE는 이 SA를 주기적으로 새로운 키로 교체(갱신)함으로써 장기간 통신으로 인한 키 노출 위험을 낮춘다.
# 통신 시 키의 실제 사용 방식
통신에 참여하는 각 하드웨어는 IKE를 통해 생성한 공유 비밀 키를 각자 가지고 있어야만 안전한 통신이 가능하다.
 키 전송 없음: 실제 데이터 통신 시에는 키가 데이터와 함께 전송되지 않는다.
 내부 사용: 발신 장치는 IKE를 통해 협상된 공유 비밀 키로 데이터를 암호화한다.
 수신 장치는 자신이 IKE를 통해 생성한 동일한 공유 비밀 키를 사용하여 데이터를 복호화한다.
