# Open Shortest Path First
## 정의
대규모 네트워크에 적합하게 설계된 대표적인 링크 상태 내부 라우팅 프로토콜
## 특징
OSPF는 [[OSI 7 계층 모델]] 중 네트워크 계층(3계층)에서 작동하며, 라우터들이 링크 상태 정보를 공유하여 경로를 결정한다.
- 다익스트라 알고리즘 사용: OSPF는 이 알고리즘을 사용하여 수집한 링크 상태 정보를 바탕으로 현재 라우터에서부터 네트워크 내의 모든 목적지까지의 가장 짧은 경로를 계산한다.
- 링크 상태: 라우터가 자신과 직접 연결된 링크의 상태(비용, 속도, 혼잡도) 정보를 [[LSA]]에 담아 네트워크 내의 모든 라우터에게 공유하며, 이를 [[LSDB]]로 취합한다.
- 홉 카운트 무제한: 홉 카운트에 제한을 두지 않고, 관리자가 정의한 [[Routing Metric]]을 사용하여 최적의 경로를 찾는다.
## AS 분할
OSPF는 AS 분할을 사용하여 라우팅의 확장성/효율성을 높인다.
모든 라우터가 네트워크 전체의 상세 정보를 다루면 부하가 커지고 계산 속도가 느려진다.
네트워크를 지역으로 분할하여, 라우터는 자신이 속한 지역의 상세 정보만 파악하고, 다른 지역의 정보는 요약된 형태로만 받아 부하를 최소화한다.
