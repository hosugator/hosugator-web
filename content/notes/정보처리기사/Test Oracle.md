# Test Oracle (테스트 오라클)
## 정의
소프트웨어 테스트에서 테스트를 수행한 결과가 올바른지(참인지)를 판단하기 위해 사용되는 모든 메커니즘 또는 정보
## 특징
테스트 결과를 수동으로 검증하는 대신 자동화된 판단의 기준으로 사용된다.
## 분류
기대하는 참 값을 제공하는 방식과 정확도에 따라 여러 유형으로 나뉜다.
### True Oracle
-   정의: 모든 입력값에 대하여 기대하는 결과를 정확히 계산하고 제공하는 오라클이다.
-   특징: 테스트 결과의 정확도가 가장 높지만, 구축하는 데 비용과 시간이 많이 소요된다.
### Sampling Oracle
-   정의: 특정한 몇 개의 입력값에 대해서만 기대하는 결과를 미리 정의하고 제공하는 오라클이다.
-   특징: 테스트 비용을 절감할 수 있으나, 샘플링되지 않은 입력값에 대한 오류는 검출할 수 없다는 한계가 있다.
### Heuristic Oracle
-   정의: 샘플링 오라클을 개선한 형태로, 특정 입력값에는 올바른 결과를 제공하고 나머지 값은 휴리스틱(추정)으로 처리하여 결과를 판단하는 오라클이다.
-   특징: 참 오라클과 샘플링 오라클의 중간 형태이며, 일부 추론을 포함하기 때문에 완벽한 정확성을 보장하지는 않는다.
### Consistency Check Oracle
-   정의: 애플리케이션에 변경(업데이트)이 있을 때, 동일한 입력에 대해 수행 전과 후의 결괏값이 동일한지 확인하여 오류를 검출하는 오라클이다.
-   특징: 회귀 테스트(Regression Test) 환경에서 유용하게 사용되며, 기능이 변경되지 않았음을 검증하는 데 초점을 맞춘다.
