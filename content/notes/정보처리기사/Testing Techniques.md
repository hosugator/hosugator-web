# 테스트 기법 (Testing Techniques)
## 정의
소프트웨어 품질을 확인하고 잠재적인 결함을 발견하기 위해 [[Test Case]]를 설계/실행하는 체계적인 방법론
## 특징
테스트 관점에 따라 다양한 기법이 존재하며 프로젝트 성격, 요구되는 품질 수준, 가용한 시간/자원에 따라 선택한다.
## 분류
대표적으로 블랙박스/화이트박스 테스팅으로 나뉜다.
### 블랙박스 테스팅
- 정의: 내부 구조나 구현을 알지 못하고 오직 사용자의 입장에서 기능의 작동성을 평가하는 기법이다.
- 특징: 요구사항 명세서, 사용자의 예상 동작을 기준으로 테스트 케이스를 설계한다.
- 종류
	- [[Cause-Effect Graph]]: 입력과 출력 간의 논리적 관계를 분석하여 테스트 케이스를 설계/시험한다.
	- 경계값 분석(Boundary Value Analysis): 오류가 경계에서 주로 발생한다는 가정 하에서 입력 값의 최소/최대/직전/직후 경계 값을 시험한다.
	- 동등 분할(Equivalent Partitioning): 입력 값을 유효/무효 범위로 나누어 각 그룹의 대표값만 시험한다.
### 화이트박스 테스팅
- 정의: 소프트웨어의 소스 코드, 내부 구조, 경로 등을 직접 확인하며 테스트 케이스를 설계/시험하는 기법이다.
- 특징: 코드의 논리적 오류, 데드 코드, 경로 누락 등을 확인하는 데 효과적이며, [[CFG]]와 밀접하게 관련된다.
- 종류
	- 구문 커버리지: 코드 내의 모든 문장을 한 번 이상 실행한다.
	- 분기/결정 커버리지: 모든 조건문의 참/거짓 분기를 한 번 이상 실행한다.
	- 조건 커버리지: 복합 조건문 내의 모든 개별 조건의 참/거짓 분기를 한 번 이상 실행한다.(분기 커버리지보다 엄격)
	- 경로 커버리지: 모든 독립된 경로를 수행하는 가장 엄격한 시험이다.
