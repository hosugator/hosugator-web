## 목적
반정규화는 이론적인 완벽성(무결성/중복 최소화)을 추구하는 정규화와 달리, 실질적인 운영 효율성을 최우선 목표로 한다.
반정규화는 실제 운영 시의 조회 속도, 성능, 관리 편의성을 우선시하는 실무적 절충 기법이다.
이는 소프트웨어 설계에서 이상적인 기능적 응집도 대신 실용적인 논리적 응집도를 선택하는 것과 유사한 맥락이다.
## 종류
### 수직 분할 (Vertical Splitting)
방식: 하나의 테이블을 컬럼(Column) 기준으로 쪼갠다.
목적: 자주 사용하는 컬럼과 잘 사용하지 않거나 큰 데이터(예: BLOB, 긴 텍스트)를 분리하여 디스크 I/O를 최소화하고 조회 성능을 높이는 데 목적이 있다.
### 수평 분할 (Horizontal Splitting, 파티셔닝)
방식: 하나의 테이블을 로우(Row) 기준으로 쪼갠다 (예: 연도별, 지역별 분리).
목적: 대용량 데이터에서 특정 범위의 데이터만 조회하도록 검색 범위를 줄여 성능을 개선하고, 대용량 데이터 관리를 용이하게 하는 데 목적이 있다.
## 테이블 분할은 왜 반정규화인가
테이블 분할은 성능 향상이라는 이점이 있음에도 불구하고, 논리적 데이터 모델의 순수성을 훼손하고 관리 복잡성을 증가시키기 때문에 반정규화로 분류된다.
### 논리적 순수성 침해
정규화가 추구하는 하나의 논리적 엔티티(개체)는 하나의 물리적 테이블로 완벽하게 존재해야 한다.
수직 분할의 경우, 논리적으로 완전해야 할 하나의 엔티티가 물리적으로 두 개 이상의 테이블로 쪼개진다.
이 행위는 논리 모델의 단순성 및 완전성이라는 논리적 순수성을 훼손한다.
### 무결성 관리 부담 증가
쪼개진 테이블의 완전한 데이터를 조회하기 위해서는 반드시 조인(JOIN) 연산이 필요해진다.
이로 인해 데이터 일관성을 유지하기 위한 관리 부담이 늘어나고, 논리적 단순성이 저하된다.
## 관련 개념
### 컬럼 중복화 (Column Redundancy)
목적: 조인 연산 없이 필요한 데이터를 바로 가져와 조인에 소요되는 시간적 비용을 절감하는 것이 주된 목적이다.
비용 감수: 데이터 중복으로 인한 저장 공간 낭비와 데이터 무결성 관리 부담 (데이터 변경 시 중복된 모든 곳을 업데이트해야 함)을 감수한다.
### 조인 (JOIN) 개념
정의: 조인은 실제 테이블을 영구적으로 합치는 것이 아니라, SQL 질의 실행 시 여러 테이블의 데이터를 엮어 사용자가 원하는 임시적인 결과 집합(Result Set)을 생성하는 연산이다.
반정규화 연관: 컬럼 중복화는 이 임시 결과 집합을 생성하는 조인 과정의 비용을 절감하기 위해 수행된다.
